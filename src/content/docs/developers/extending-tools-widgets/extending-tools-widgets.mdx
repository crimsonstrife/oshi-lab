title: Extending Tools & Widgets

This page is for contributors maintaining the lab’s built-in tools and widget library.

## Adding a new tool

Tools are auto-registered from:

- `tools/defs/*.tool.js`

A tool module should `export default` an object with:

- `id` (unique)
- `name`
- `keywords` (optional)
- `order` (optional)
- `render(panel)` — render the tool UI into the provided element

The tools modal reads definitions via `import.meta.glob('./defs/*.tool.js', { eager: true })`.

### Marked snippets (recommended)

If your tool generates a “block” of CSS/HTML that users may want to re-generate later, wrap it with stable markers so the tool can **update in place**:

- `buildMarkedSnippet({ kind, blockId, version, body })`
- `upsertMarkedSnippet(textareaEl, kind, blockId, body, version)`

Marker examples:

```css
/* oshilab:begin block=theme-vars v=1 */
...
/* oshilab:end block=theme-vars */
```

```html
<!-- oshilab:begin block=schedule-builder/html v=1 -->
...
<!-- oshilab:end block=schedule-builder/html -->
```

**Block ID conventions:**

- Prefer namespaced IDs for multi-part output:
  - `widget/<id>/html`, `widget/<id>/css`
  - `schedule-builder/html`, `schedule-builder/css`
  - `cssblock/<id>`
- Bump `version` when you make a breaking change to a block’s structure.

## Adding a new widget

Widgets are auto-registered from:

- `tools/widgets/defs/*.widget.js`

A widget definition should include:

- `id` (unique)
- `name`
- `description` (optional)
- `keywords` (optional)
- `order` (optional)
- `version` (optional, default `1`)
- `html` (string)
- `css` (string)
- `fields` (optional array)

Most widgets load raw files:

```js
import html from './your-widget/your-widget.html?raw';
import css from './your-widget/your-widget.css?raw';
```

### Field schema

Each field supports:

- `id`
- `label`
- `defaultValue`
- `type` (optional, e.g. `textarea`)
- `allowHtml` (optional boolean)
- `placeholder` (optional)

The Widget Inserter replaces `{{fieldId}}` placeholders in `html`/`css`.

## Adding a new CSS block

CSS blocks are auto-registered from:

- `tools/cssBlocks/defs/*.block.js`

A block definition should include:

- `id` (unique)
- `name`
- `description` (optional)
- `keywords` (optional)
- `order` (optional)
- `version` (optional, default `1`)
- `css` (string), usually imported from a raw `.css` file

The CSS Blocks tool reads definitions via `import.meta.glob('./defs/*.block.js', { eager: true })`.
